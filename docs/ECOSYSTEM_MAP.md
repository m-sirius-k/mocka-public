# MoCKA Insight System Map

--------------------------------
ENGLISH VERSION
--------------------------------

## Overview

The MoCKA Insight System is composed of multiple repositories that together form a verifiable AI architecture.

Each repository represents a structural layer of the system.

The Insight System is designed so that architecture, documentation, and verification systems remain observable and reproducible.

---

## Repository Structure

MoCKA

Core orchestration and verification infrastructure.

Includes:

Research Gate  
Doctor integrity system  
Canonical research mapping  
Artifact generation

---

MoCKA-KNOWLEDGE-GATE

Institutional memory layer.

Preserves reasoning traces, hypothesis evolution, and structured research knowledge.

---

mocka-civilization

Conceptual layer describing the civilization architecture of MoCKA.

Contains design philosophy, structural principles, and long-term architectural ideas.

---

mocka-transparency

Public observability layer.

Designed to expose system structure and audit trails for external evaluation.

---

mocka-external-brain

External reasoning and knowledge storage layer.

Supports knowledge continuity beyond individual runtime sessions.

---

mocka-core-private

Private operational layer.

Contains internal tools and operational infrastructure not intended for public exposure.

---

## Insight System Verification

The Insight System integrity is verified by two main systems.

Doctor

Performs structural scans of repositories and generates verification artifacts.

Research Gate

Runs structured experiments validating system integrity and documentation structure.

---

## Verification Flow

Typical verification flow.

Developer updates repository structure

Doctor scan executes

Artifact generated

Research Gate runs experiments

Integrity confirmed

---

## Purpose

The purpose of the Insight System is to demonstrate that complex AI architectures can maintain transparency and structural verification at scale.
